# Technology Stack Justification – Vocabulary Quiz Generator (Python Console)

เอกสารฉบับนี้อธิบายเหตุผลในการเลือกใช้เทคโนโลยีต่าง ๆ สำหรับโครงการ  
“Vocabulary Quiz Generator” ซึ่งพัฒนาด้วยภาษา Python ในรูปแบบ Console Application

---

## 1. Programming Language: Python

**เทคโนโลยีที่เลือกใช้:**  
- ภาษาโปรแกรม: **Python 3.x**

**เหตุผลที่เลือกใช้:**

1. **เรียนรู้ง่าย และเหมาะกับงานด้านการศึกษา**  
   Python มีไวยากรณ์ที่อ่านเข้าใจง่าย (Readable) เหมาะกับโครงการในวิชาวิศวกรรมซอฟต์แวร์  
   ทำให้โฟกัสไปที่การออกแบบโครงสร้างโปรแกรม และ SDLC มากกว่ารายละเอียด syntax

2. **รองรับการจัดการไฟล์และข้อความได้ดี**  
   โครงการนี้ต้องอ่านข้อมูลจากไฟล์คำศัพท์ (Text/CSV) และประมวลผล String  
   Python มีฟังก์ชันมาตรฐานด้าน File I/O และ String handling ที่ใช้งานง่าย เช่น `open()`, `readline()`, `split()`

3. **Standard Library ครอบคลุมความต้องการของโปรเจกต์**  
   ฟังก์ชันที่ต้องใช้ เช่น การสุ่ม (random), การจัดการรายการ (list)  
   สามารถใช้ได้จาก Standard Library โดยไม่ต้องติดตั้ง Library เพิ่มเติม

4. **ทำงานข้ามแพลตฟอร์ม (Cross-platform)**  
   โปรแกรมสามารถรันได้ทั้งบน Windows, macOS และ Linux ที่ติดตั้ง Python 3  
   ทำให้การทดสอบและการส่งงานให้ผู้สอนทำได้สะดวก

---

## 2. Runtime Environment & Platform

**สภาพแวดล้อมที่คาดว่าจะใช้งาน:**

- ระบบปฏิบัติการ: Windows 10/11 (หรือเทียบเท่า)
- Python Interpreter: Python 3.x
- รูปแบบการใช้งาน: รันผ่าน Terminal / Command Prompt (Console-based)

**เหตุผล:**

1. **รองรับในเครื่องนักศึกษาส่วนใหญ่**  
   นักศึกษาสามารถติดตั้ง Python 3 และรันโปรแกรมผ่าน Command Prompt / Terminal ได้เอง

2. **ไม่ต้องพึ่งพา Web Server หรือ Framework ภายนอก**  
   ลดความซับซ้อนในการ Deployment และทำให้การทดสอบบนเครื่องส่วนตัวง่ายขึ้น

---

## 3. Data Storage: Text/CSV File

**เทคโนโลยีที่ใช้จัดเก็บข้อมูลคำศัพท์:**

- รูปแบบไฟล์: `.txt` หรือ `.csv`
- โครงสร้างข้อมูล: ข้อมูลต่อ 1 บรรทัด เช่น  
  `word,correct_answer,choice1,choice2,choice3,choice4`

**เหตุผลที่เลือกใช้ File-based Storage แทน Database:**

1. **ตรงกับโจทย์วิชาและขอบเขตโครงการ**  
   โครงการเน้นการอ่านและประมวลผลข้อมูลจากไฟล์ภายนอก จึงไม่จำเป็นต้องใช้ฐานข้อมูลจริง

2. **ลดความซับซ้อนในการติดตั้งและใช้งาน**  
   ไม่ต้องติดตั้ง Database Server เช่น MySQL หรือ PostgreSQL  
   ง่ายต่อการรันโครงการบนเครื่องของอาจารย์

3. **เพียงพอกับฟังก์ชันของระบบ**  
   จำนวนคำศัพท์ในไฟล์มีขนาดไม่มาก  
   การโหลดข้อมูลทั้งหมดเข้าสู่หน่วยความจำ (list/dict) สามารถทำได้สบายใน Python

---

## 4. Libraries & Modules

**Standard Libraries ที่ใช้:**

1. `random`  
   - ใช้สำหรับสุ่มลำดับคำถาม และการสลับลำดับตัวเลือกคำตอบ  
   - เหตุผล: ทำให้ข้อสอบไม่ซ้ำเดิมทุกครั้ง เพิ่มประสิทธิภาพในการฝึกคำศัพท์

2. `os` หรือ `pathlib` (ถ้าจำเป็น)  
   - ใช้ตรวจสอบการมีอยู่ของไฟล์ หรือจัดการ path ของไฟล์คำศัพท์  
   - เหตุผล: เพิ่มความทนทานของโปรแกรม และช่วยจัดการ Error กรณีไม่พบไฟล์

3. ฟังก์ชันพื้นฐานจาก Python เช่น `open()`, `input()`, `print()`  
   - ใช้สำหรับอ่านไฟล์, รับ input จากผู้ใช้, และแสดงผลบนหน้าจอ Console

**ไม่มีการใช้ External Library เพิ่มเติม**  
เพื่อให้การติดตั้งและการรันโปรแกรมง่ายที่สุด ผู้ใช้เพียงแค่ติดตั้ง Python 3 ก็สามารถรันโปรแกรมได้

---

## 5. Version Control: Git + GitHub

**เทคโนโลยีที่ใช้:**

- Version Control: **Git**
- Remote Repository: **GitHub (Public Repository)**

**เหตุผล:**

1. **สอดคล้องกับเกณฑ์การประเมินของวิชา**  
   งาน Final Project ต้องส่งผ่าน GitHub Public Repository และใช้โครงสร้างโฟลเดอร์ตาม SDLC

2. **รองรับการติดตามการเปลี่ยนแปลงของโค้ด**  
   Git ช่วยให้สามารถดูประวัติการแก้ไข (commit history) และย้อนกลับไปยังเวอร์ชันก่อนหน้าได้

3. **ความเป็นมืออาชีพในการพัฒนา Software**  
   การใช้ Git และ GitHub เป็นมาตรฐานในอุตสาหกรรม  
   โครงการนี้จึงเป็นการฝึกใช้เครื่องมือในโลกจริงไปด้วยในตัว

## 6. เหตุผลในการไม่เลือกเทคโนโลยีอื่น (Alternatives Considered)

1. **ไม่ใช้ Web Framework (เช่น Django/Flask)**  
   - เกินความจำเป็นของโปรเจกต์  
   - เพิ่มความซับซ้อนด้าน Deployment และ Learning Curve  
   - ไม่ตรงกับโจทย์ที่เน้น Console Application และการอ่านไฟล์

2. **ไม่ใช้ GUI Framework (เช่น Tkinter, PyQt)**  
   - แม้จะช่วยให้ UI ดูสวยงามขึ้น แต่จะเพิ่มภาระในการออกแบบและทดสอบ  
   - โครงการนี้ให้ความสำคัญกับ Logic, SDLC และการจัดการไฟล์มากกว่า UI ขั้นสูง

3. **ไม่ใช้ Database (เช่น SQLite, MySQL)**  
   - การเก็บข้อมูลคำศัพท์ในไฟล์เพียงพอแล้ว  
   - ไม่ต้องจัดการ Schema, Connection, และคำสั่ง SQL  
   - ลดเวลาและความซับซ้อนในการพัฒนา

---

## 7. สรุปภาพรวมของ Technology Stack

โครงการ “Vocabulary Quiz Generator” เลือกใช้เทคโนโลยีที่:

- **เรียบง่าย** (Python + Text File + Console)
- **ติดตั้งง่าย** (ใช้แค่ Python 3 และ Git)
- **ตอบโจทย์ทางวิชาการ** (เน้น Logic, SDLC, การอ่านไฟล์, การสุ่ม, และการทดสอบ)
- **รองรับการส่งงานอย่างเป็นระบบ** ผ่าน GitHub Public Repository
